# Typing Rush MVP Tasks

## 目标
基于 [`spec.md`](D:\WorkPlace\Codex\typing-rush\spec.md) 完成一个可在 Windows 上运行的程序员打字竞速桌面应用 MVP。首版范围只覆盖本地单机、代码片段打字、结果统计、本地持久化和 Tauri 打包。

## 里程碑

### M1. 项目骨架与基础运行链路
- 初始化 `Vite + React + TypeScript` 前端工程。
- 初始化 Tauri 桌面壳，保证本地开发与打包路径成立。
- 建立目录结构：`src/engine`、`src/content`、`src/render`、`src/storage`、`src/ui`、`src/types`。
- 建立全局主题、字体、基础布局和页面路由/状态容器。
- 定义核心 TypeScript 类型：
  - `Language`
  - `Difficulty`
  - `RunDuration`
  - `Snippet`
  - `RunConfig`
  - `RunStats`
  - `ResultRecord`

### M2. 单局主流程闭环
- 实现顶层应用状态机：
  - `menu`
  - `countdown`
  - `playing`
  - `paused`
  - `result`
- 实现菜单页：
  - 时长选择：`5m` / `10m`
  - 语言选择：`JavaScript` / `TypeScript` / `Python`
  - 难度选择：`Easy` / `Medium` / `Hard`
  - 声音开关
- 实现 3 秒倒计时页面与状态切换。
- 实现单局结束后的结果页骨架。
- 验收标准：
  - 能从菜单进入倒计时，再进入结果页
  - 5 分钟模式的计时闭环可跑通

### M3. 输入判定与实时统计
- 实现键盘输入控制层，稳定接收：
  - 可见字符
  - `Backspace`
  - `Enter`
  - `Tab`
- 定义字符判定规则：
  - 正确字符推进光标
  - 错误字符即时标记
  - `Backspace` 允许修正，但错误数保留统计
- 实现局内实时指标计算：
  - `Raw WPM`
  - `WPM`
  - `Accuracy`
  - `Combo`
  - `Max Combo`
  - `Completed Snippets`
  - `Score`
- 定义停顿惩罚和稳定性加成的最小可行算法。
- 验收标准：
  - 输入和统计同步更新
  - 错误输入有明确反馈
  - 结果页能展示本局指标

### M4. Canvas 渲染主练习区
- 实现代码片段主渲染器，负责：
  - 文本布局
  - 当前字符高亮
  - 错误字符着色
  - 光标显示
  - 多行片段滚动/换段
- 统一 `Tab` 的内容规范，渲染和判定使用相同的空格展开策略。
- 建立性能基线，确保 10 分钟模式下稳定。
- 验收标准：
  - 1 到 8 行片段都能稳定显示
  - 连续输入时无明显闪烁或光标错位

### M5. 内容系统与连续推题
- 建立题库数据结构和文件组织方案。
- 为三种语言和三种难度建立初始题库。
- 每个语言每个难度至少准备 20 段，目标总量约 180 段。
- 实现筛选与随机队列：
  - 按语言筛选
  - 按难度筛选
  - 避免短时间重复同片段
- 实现片段完成后的无缝切换。
- 验收标准：
  - 整局不会空题
  - 10 分钟模式重复率可控

### M6. 本地存档与结果系统
- 设计 `settings` 和 `results` 的本地存储结构。
- 实现默认设置持久化：
  - 默认时长
  - 默认语言
  - 默认难度
  - 声音开关
- 实现结果记录持久化：
  - 最近 20 条成绩
  - 各模式最佳成绩
  - 各语言最佳成绩
- 在结果页展示：
  - 本局结果
  - 历史最佳
  - 最近成绩趋势
- 验收标准：
  - 重启后设置和成绩仍可读取

### M7. 暂停恢复、提示与打磨
- 实现窗口失焦自动暂停。
- 实现点击恢复继续输入。
- 实现每分钟一次的轻量阶段提示。
- 增加最小必要音效：
  - 命中
  - 错误
  - 倒计时结束
  - 局终结算
- 做 10 分钟模式专项稳定性检查。
- 验收标准：
  - 失焦暂停可靠
  - 10 分钟模式无明显状态漂移和掉帧

### M8. 桌面打包与交付
- 接通 Tauri 构建配置。
- 验证 Windows 打包产物可运行。
- 补齐图标、窗口标题、基础应用元信息。
- 补最小 README 与启动说明。
- 验收标准：
  - 可生成 Windows 桌面应用安装包或可执行产物

## 任务清单

### 核心代码任务
- 建立共享类型与常量文件。
- 建立应用状态机和 reducer/store。
- 实现游戏时钟和 countdown timer。
- 实现输入缓冲与字符判定器。
- 实现实时统计聚合器。
- 实现分数计算器。
- 实现片段队列选择器。
- 实现 Canvas 文本渲染器。
- 实现本地存储适配器。
- 实现菜单、HUD、结果页、暂停提示层。

### 内容任务
- 确定题库文件格式，推荐 JSON 或 TS 常量模块。
- 规范 snippet 文本中的换行和缩进。
- 为每条 snippet 标注 `language`、`difficulty`、`tags`。
- 先覆盖高频训练点：
  - 括号与配对符号
  - 条件与循环
  - 函数声明与调用
  - 字符串和模板字符串
  - 对象/数组
  - 异步与异常处理

### 测试任务
- 为计分、判定、队列算法补单元测试。
- 为状态机流转补单元测试。
- 为存储读写补测试。
- 为关键 UI 流程补集成测试：
  - 菜单到开局
  - 倒计时到游戏中
  - 时间结束到结果页
  - 失焦暂停与恢复

## 建议实施顺序
1. 先完成 M1 到 M3，确保游戏逻辑闭环成立。
2. 再完成 M4 到 M5，解决“能玩”和“可持续玩”。
3. 然后完成 M6 到 M7，补持久化和长局体验。
4. 最后完成 M8，进入桌面交付。

## Definition of Done
- 用户可以在 Windows 桌面环境启动应用。
- 用户可以选择时长、语言、难度并开始一局。
- 应用能持续推送代码片段并逐字符判定。
- HUD 能实时展示核心指标。
- 结束后能看到结果页与历史数据。
- 重启后设置和成绩保留。
- 窗口失焦时暂停，恢复后继续。
- 10 分钟模式无明显性能问题。
